Spark纵览

Apache Spark是一个快速通用的集群计算系统。它为Java，Scala，Python以及R提供了一组 高度抽象的API，并且还提供了用于支持通用执行图算法的一个优化过的引擎。它还支持一大组 高层次的工具，包括用于SQL和结构化数据处理的Spark SQL， 用于机器学习的MLlib，用于图处理的GraphX, 以及Spark Streaming。

下载

您可以通过项目中的下载页来获得Spark程序包。 这篇文档用于Spark版本1.5.1。Spark使用了Hadoop客户端库来处理HDFS和YARN。 下载中包含了与一些流行Hadoop版本的整合好的Spark包。用户同样也可以自行选择下载任意版本”免费Hadoop”二进制包， 然后在其上运行Spark。如何扩展Spark的类路径.

如果您打算自行从源代码来构建Spark，请看构建Spark.

Spark可以运行在Window以及其他类Unix系统上(比如Linux和Mac OS)。想在本地单机上运行非常容易，您所需要的 仅仅是安装Java并配置其到您的系统PATH中，或者配置JAVA_HOME系统变量来指向您的Java安装路径。

Spark需要运行在Java 7+，Pythong 2.6+以及R3.1+之上。对于Scala API，Spark 1.5.1 使用 Scala 2.10。您需要使用一个兼容的Scala版本(2.10.x)。

运行例子程序以及Spark shell

Spark本身包含一些例子程序。Scala，Java，Python以及R例子都位于examples/src/main目录下。 您可以在Spark安装个目录中使用bin/run-example <class> [params]来运行任意的Java或Scala例子 程序。(当然在幕后，为了运行程序，这会调用更为通用的spark-submit脚本。 例如，

./bin/run-example SparkPi 10

您同样可以通过一个修改版本的Scala shell来交互式的运行Spark。这是一种更好的学习该框架的方式。 You can also run Spark interactively through a modified version of the Scala shell. This is a

./bin/spark-shell --master local[2]

--master选项用于声明一个分布式集群的主URL， 或者使用local来在本地的一个线程中运行，或使用local[N]来在多个本地线程中运行。您可以先使用 local来尝试一下。要查看其他选项，请使用--help选项来运行Spark shell。

Spark同样提供了一组Python API。要使用Python解释器来交互式的运行Spark，请使用 bin/pyspark:

./bin/pyspark --master local[2]

Python例子程序也有提供。例如，

./bin/spark-submit examples/src/main/python/pi.py 10

从1.4开始，Spark同样也提供了一组实验性的R API(只包含了DataFrames API)。 要使用R解释器来交互式的运行Spark，请使用bin/sparkR:

./bin/sparkR --master local[2]

R例子程序也有提供。例如，

./bin/spark-submit examples/src/main/r/dataframe.R

在集群上运行

Spark集群模式概览解释了需要在集群上运行Spark的关键的概念。Spark可以自行在其上运行，也可以 通过一些现存的集群管理器来运行。目前提供了以下一些选项来进行部署：

  • Amazon EC2: 我们的EC2脚本可以允许您在五分钟之内启动一个EC2集群
  • Standalone Deploy Mode: 最简单的方式来在一个私有集群中部署Spark
  • Apache Mesos
  • Hadoop YARN
